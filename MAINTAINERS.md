# Maintainers

This file names the people and teams responsible for project stewardship,
review, releases, and security coordination.

## Current maintainers

| Name / team | GitHub | Role | Notes |
|---|---|---|---|
| XOps Labs maintainers | `@xops-labs/maintainers` | Project lead / maintainer team | Review, release, security coordination. |

## Responsibilities

Maintainers are expected to:

- Review pull requests in areas they understand.
- Keep public docs honest about current implementation.
- Protect the Apache-2.0 OSS boundary and avoid bundling external rule-pack
  overlay material.
- Keep CI, release, and security reporting paths working.
- Respond to security reports according to [SECURITY.md](SECURITY.md).

## Becoming a maintainer

Maintainer status is earned through sustained, constructive contributions such
as code review, detector/rule work, documentation, release help, or user support.
A maintainer nomination should be opened publicly unless there is a privacy or
conduct reason not to.

## Inactivity and offboarding

Maintainers who are inactive for an extended period may be moved to emeritus
status after a public notice or private check-in. Access can be restored later
when active participation resumes.

## Conflict of interest

Maintainers should disclose relationships that could materially affect decisions
about scanner coverage, rule packs, cryptography vendors, or security products.
Disclosure does not prevent participation; it helps reviewers understand context.
